Grocery Run

Over the course of two week-long sprints, two colleagues and I built out this fullstack grocery list-building application. In this application Users can search local grocery stores for product. Users can add these results to their upcoming "trip" where the price and distance/location info for each item will be displayed. Users can also make and add a grocery list to their "trip" where each item will be auto selected from the search results based on relevance/price/distance. We built out the backend using JavaScript/Node.js and the MongoDB DBMS. Our backend was made to securely store and utilize data integrated from two external APIs (Kroger API & Albertsons API) to collect data

Tower

This Full-Stack Application was built over a one-week sprint during which I focused on honing in my Vue.js skills while also working to better execute complicated many-to-many relationships with Node.js. In this application Users can create, view, and attend "tower events". Each event has visual indicators for event cancellation or an event reaching max capacity. Users also have the ability to leave comments on other users' publicly posted events. On the client side, I implemented VueJs with Vue-Router. On the server-side, I used Express with Node.js and MongoDB with Mongoose to manage my database and handle both one-to-many and many-to-many data relationships. In addition, I utilized identity management using Auth0.
